# RPNL / Total PnL Reconciliation — UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the four UI changes in FEDEV-3758 so trade-history RPNL reconciles with the General Performance Details panel after the paired squid-side PR is deployed.

**Architecture:** Four independent changes across trade-history rendering, account dashboard debug tooltip, leaderboard tooltip, and account dashboard PnL tooltip. Changes share a ticket but no runtime state. Spec: `docs/superpowers/specs/2026-04-20-rpnl-reconcile-ui-design.md`.

**Tech Stack:** React, TypeScript, lingui (i18n), vitest (unit tests), Apollo Client (GraphQL against Subsquid indexer), Tailwind CSS, GMX SDK workspace package.
